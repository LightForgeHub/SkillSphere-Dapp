# Issue #335: Developer Webhook Management Console UI

**Branch Name:** `feature/335-webhook-management-console`
**Commit Message:** `feat: add developer webhook management dashboard UI - in-progress`

## Overview
Add a developer dashboard panel to input URLs for external services to register endpoint URLs for on-chain events.

## Acceptance Criteria
- Form fields to input: Webhook Target URL, Secret Token
- Event list selection checkboxes (Session Started, Session Closed, Dispute Opened)

## Planned Implementation

### 1. Component Architecture
- **WebhookManagement Component**: Main dashboard component
  - Webhook registration form
  - List of registered webhooks
  - Event configuration panel

- **WebhookForm Component**: Form for adding/editing webhooks
  - URL input validation
  - Secret token field (with show/hide)
  - Event selection checkboxes

- **WebhookList Component**: Display registered webhooks
  - Status indicators (active/inactive)
  - Edit/delete actions
  - Webhook details

### 2. Form Fields Implementation
- **Webhook Target URL**: Validated URL input
  - Protocol validation (HTTPS required)
  - Domain validation
  - Real-time format checking

- **Secret Token**: Secure input field
  - Password masking with show/hide toggle
  - Token generation option
  - Copy to clipboard functionality

- **Event Selection**: Checkbox group
  - Session Started
  - Session Closed  
  - Dispute Opened
  - Expandable for future events

### 3. State Management
- Form validation state
- Webhook configuration state
- Loading/error states for API calls
- Local storage for draft configurations

### 4. API Integration Structure
- **Webhook Registration**: POST /api/webhooks
- **Webhook Listing**: GET /api/webhooks  
- **Webhook Update**: PUT /api/webhooks/:id
- **Webhook Deletion**: DELETE /api/webhooks/:id
- **Webhook Test**: POST /api/webhooks/:id/test

### 5. UI/UX Design
- Developer-focused dashboard layout
- Clear form validation feedback
- Intuitive event selection
- Responsive design
- Success/error notifications
- Confirmation dialogs for destructive actions

### 6. Security Considerations
